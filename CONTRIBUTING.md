# Contributing

1. Fork the repository and create a focused branch.
2. Install dependencies with `pip install -r requirements-dev.txt`.
3. Add or update tests for behavior changes.
4. Run `ruff check .` and `pytest`.
5. Open a pull request explaining the problem, solution, and screenshots when UI changes are involved.

Keep pull requests small, avoid committing database files, and never include real applicant or employer personal data in test fixtures.
